This is exactly where Pixeltable earns its keep. It gives you a declarative table where derived values are computed columns — define the transform once and it stays in sync as data arrives. No DAGs to wire, no batch jobs to babysit.

Semantic search over text with an embedding index

Retrieval is the backbone of RAG. Pixeltable lets you attach an embedding index to a column and run similarity search with a single call -- the embeddings are computed and indexed for you as data arrives.

import pixeltable as pxt

pxt.drop_dir("demo", force=True)
pxt.create_dir("demo")

t = pxt.create_table("demo.docs", {"id": pxt.Int, "text": pxt.String})
t.insert([
    {"id": 1, "text": "Pixeltable stores and transforms multimodal data."},
    {"id": 2, "text": "Cats are small domesticated carnivorous mammals."},
    {"id": 3, "text": "Vector search finds semantically similar documents."},
    {"id": 4, "text": "The Eiffel Tower is located in Paris, France."},
])

# A simple, deterministic, offline embedding so the demo needs no model files.
@pxt.udf
def toy_embed(s: str) -> pxt.Array[(16,), pxt.Float]:
    import numpy as np
    v = np.zeros(16, dtype=np.float32)
    for tok in s.lower().split():
        v[hash(tok) % 16] += 1.0
    n = np.linalg.norm(v)
    return v / n if n else v

t.add_embedding_index(column="text", string_embed=toy_embed)

sim = t.text.similarity("searching for similar text with vectors")
print(t.order_by(sim, asc=False).select(t.id, t.text, score=sim).limit(2).collect())

Run it and you get real output — here's the actual result captured from this exact snippet (ran in 5.73s):

id                                                 text     score
0   3  Vector search finds semantically similar documents.  0.516398
1   4        The Eiffel Tower is located in Paris, France.  0.182574

That's not a mockup. The code above executed against Pixeltable and produced that table. Swap the toy pieces (a UDF, a local embedding) for production ones — a real model endpoint, a managed embedding — and the shape of your code doesn't change. That's the point.

Takeaways

  • Declarative beats imperative for data + AI: say what you want computed, not how to schedule it.
  • Computed columns stay in sync automatically as new rows land.
  • Multimodal is first-class — images, video, audio, and text live in the same table as your scalars.

Try it yourself

pip install pixeltable
