A hands-on, runnable walkthrough using image — with real, executed output.

How we index images for RAG keeps showing up in the feeds I track — and for good reason. The hard part is rarely the idea; it's the plumbing: wiring data, transforms, and models together without it turning into a pile of glue scripts. Recurring themes around this topic: image, rag, here, claude, code, anthropic.

This is exactly where Pixeltable earns its keep. It gives you a declarative table where derived values are computed columns — define the transform once and it stays in sync as data arrives. No DAGs to wire, no batch jobs to babysit.

A multimodal image table with computed columns

Pixeltable treats images as first-class data. Here we build a table of images, then add computed columns that derive new values automatically -- no manual loops, no glue code. New rows get the same transforms for free.

import pixeltable as pxt
from PIL import Image

# A fresh, isolated workspace.
pxt.drop_dir("demo", force=True)
pxt.create_dir("demo")

# Generate a few local images so the example is fully offline.
import tempfile, os
tmp = tempfile.mkdtemp()
paths = []
for i, color in enumerate([(220, 50, 50), (50, 180, 90), (60, 90, 220)]):
    p = os.path.join(tmp, f"img_{i}.png")
    Image.new("RGB", (256 + i * 32, 192), color).save(p)
    paths.append(p)

# Declare a table with an image column and insert the rows.
t = pxt.create_table("demo.images", {"img": pxt.Image, "label": pxt.String})
t.insert([{"img": p, "label": l} for p, l in zip(paths, ["red", "green", "blue"])])

# Computed columns: derived automatically and kept in sync on insert.
t.add_computed_column(width=t.img.width)
t.add_computed_column(height=t.img.height)
t.add_computed_column(thumb=t.img.resize([96, 96]))

print(t.select(t.label, t.width, t.height).collect())

Run it and you get real output — here's the actual result captured from this exact snippet (ran in 9.84s):

label  width  height
0    red    256     192
1  green    288     192
2   blue    320     192

That's not a mockup. The code above executed against Pixeltable and produced that table. Swap the toy pieces (a UDF, a local embedding) for production ones — a real model endpoint, a managed embedding — and the shape of your code doesn't change. That's the point.

Takeaways

  • Declarative beats imperative for data + AI: say what you want computed, not how to schedule it.
  • Computed columns stay in sync automatically as new rows land.
  • Multimodal is first-class — images, video, audio, and text live in the same table as your scalars.

Try it yourself

pip install pixeltable
